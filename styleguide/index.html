Introduction

The Pension Master User Manual is generated using a static website generator called Nanoc. This allows all the pages to be written using Markdown or in this case a variant called Kramdown.

Because each page is converted into HTML markup this cleans up the pages considerably.

Headings

Headings are indicated by the ## at the beginning of the line. The main page heading is a <h1>, which is the taken from the title of the markdown file. Other headings then start at ## with any sub-headings being ### etc.

Paragraphs

Paragraph tags <p> and </p> are not required as the Markdown processor takes the blank lines around the text to indicate paragraphs.

Lists

For unordered lists start each line with * :

* Item 1
  * Item a
  * Item b
  * Item c
* Item 2
* Item 3
  • Item 1
    • Item a
    • Item b
    • Item c
  • Item 2
  • Item 3

For ordered lists start each line with 1. :

1. Item 1
   1. Item a
   1. Item b
   1. Item c
1. Item 2
1. Item 3
  1. Item 1
    1. Item a
    2. Item b
    3. Item c
  2. Item 2
  3. Item 3

Links

These have been left as standard HTML tags:

<a href="/pension/system_administration/gn_user/">Users</a>

New links could be written using the Markdown syntax:

[Users](/pension/system_administration/gn_user/)

Tables

Tables are created just using pipes as column markers but they need a little extra code to style them.

  1. The first line in pipes below indicates the table headers.
  2. The second line shows that the columns are to be left justified.
  3. Any subsequent lines are data lines.

The Bootstrap stylesheet uses 12 columns across the screen. The col-md-6 class indicates that the table should span half of the screen width.

<div class="row"><div class="col-md-6">

{:.table .table-condensed}
| Field | Default Value |
|:------|:--------------|
|Payment Frequency|12 - Monthly|

</div></div>
Field Default Value
Payment Frequency 12 - Monthly

Code

Blocks of code, such as the examples of lines in Formula, are wrapped in two lines of 5 tilde characters.

~~~~~
@ln1 = @ln2 goto @ln3
~~~~~

@ln1 = @ln2 goto @ln3

Escape

Escape pipes that need to be displayed need to be escaped using \.

Keys

Key presses are wrapped in <kbd></kbd> tags like Return.

Next and Previous key presses have an arrow to the left of the text using:

'<span class="glyphicon glyphicon-arrow-left"></span> Previous' or '<span class="glyphicon glyphicon-arrow-right"></span> Next'

‘ Previous’ or ‘ Next’

Wording

Drop-down List

These are the fields that just display a list of values underneath the field e.g. Database on the Logon screen.

If the options are static and don’t come from a database table then write:

*SEX* - Select one of the following options from the drop-down list:

* Male
* Female

If the options are from a database table then write:

Select a currency from the drop-down list.

Lookup List

Any fields that have a lookup icon which then opens a lookup list then write:

Enter a valid broker code, or click on the lookup icon for a list of broker codes.

Client specific areas
