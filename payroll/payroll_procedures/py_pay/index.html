Introduction

This program allows the user to run a payroll for a specified scheme. From this program the user can print the payslips and payment summary reports for the members of the scheme.

Only users set up as a System Administrator can access this program.

If another user is running either the Backup Data program, the Restore Data program, or the Year End program, then you are unable to process the Payroll Run until the other program has completed.

There must be Government Legislation details set up for the tax year, you wish to run the Payroll in. If there isn’t, then the user will be informed of this when they attempt to run a payroll.

The user will be allowed to use the previous tax year’s Government Legislation details, however, they must update the Government Legislation program with the new tax year details as soon as they receive them.

The following options and screen appear when this program is selected:

  • Run
  • Submit
  • Check
  • Exit

Payroll Run Screen

Run A Payroll

Click ‘Run’.

SCHEME CODE - Enter the scheme code required. To get a list of valid scheme codes click on the lookup icon. When a valid scheme code has been entered the scheme description will be displayed. This field must be entered.

CATEGORY CODE - Enter the category code required. To get a list of valid category codes click on the lookup icon. When a valid category code has been entered the category description will be displayed. This field should be left blank if you wish to run the payroll for every category of the scheme.

RUN DATE - Enter the run date of the backup in the format dd/mm/yyyy or ddmmyyyy. This field is defaulted as today’s date, however, this can be altered to another date. Click on the calendar icon to display a calendar on the screen. This field must be entered.

Click ‘OK’ to proceed with the payroll run. You can click ‘Cancel’ to return to the payroll menu.

If the run date entered is before the date of the last payroll run, an error message will be displayed informing the user that there are no members ready for payroll together with the date the payroll was last run. They will then be asked to input a different date in the run date field.

If the run date entered falls inside a new tax year and the last run date falls in the previous tax year, Payroll Master checks if the Year End procedure has been carried out. If it hasn’t then the user is informed of this and they will not be allowed to continue with the payroll until the year end procedure has been completed.

If the scheme pays their members in advance then members whose date pension commenced is after the run date entered, but before the next payroll run date, then they will be included in the current payroll run.

The following message is now displayed to the screen:

Do you wish to continue running the Payroll?

If ‘Yes’ is selected, then the payroll run will continue. If ‘No’ is selected, then the program returns to the payroll menu.

The following screen is displayed:

Payroll Run Screen

The members of the scheme selected for the payroll run are displayed in this screen. You can use the arrow buttons to scroll through the selected members. Some of the fields can be adjusted as follows:

CODE - The payment element code is displayed to this field. You cannot enter into this field.

DESCRIPTION - The payment element description is displayed to this field. You cannot enter into this field.

TYPE - The type of payment element is displayed to this field. This will either be Addition or Deduction. You cannot enter into this field.

TAX - The taxability of the amount is displayed to this field. This will either be Pre or Post. You cannot enter into this field.

MULTIPLE - The multiple of the member’s payment element rate is displayed to this field. This is defaulted to 1. The user can adjust this value accordingly.

RATE - The rate of the member’s payment element is displayed to this field. This is retrieved from the Payment Element screen in Payroll Members. The PENSION code rate is calculated by Payroll Master by dividing the member’s pension amount by the member’s payment frequency e.g. Monthly = 12, Weekly = 52 etc. The user can adjust this value accordingly.

AMOUNT - The amount of the member’s payment element is displayed to this field. This is calculated by Payroll Master by multiplying the ‘Multiple’ value by the ‘Rate’ value. You cannot enter into this field.

TAX CODE - The tax code of the member is displayed to this field. You cannot enter into this field.

GROSS PAY - The amount of the member’s gross pay is displayed to this field. This is calculated by Payroll Master by adding the pension amount with any other Addition type payment elements. You cannot enter into this field.

PAYE - The amount of PAYE the member is to pay is displayed to this field. This is calculated by Payroll Master. You cannot enter into this field.

NIC - The amount of nic the member is to pay is displayed to this field. This is calculated by Payroll Master. You cannot enter into this field.

NET PAY - The amount of net pay the member is to be paid is displayed to this field. This is calculated by Payroll Master. You cannot enter into this field.

Click ‘OK’ to continue with the payroll run. Click ‘Cancel’ to return to the payroll menu.

Exceptions Report

When a payroll is run, all the members in the scheme must have all the information required in Pension Master and Payroll Master. If any vital information does not exist, e.g. Tax Code, Payment Frequency, Pension Amount or Bank Details, then an exceptions report is generated.

The following window will pop up:

Exceptions Found Window

The information must be entered onto Pension Master and Payroll Master before the Payroll Run can be successfully carried out.

Information Report

When a payroll is run, if any members in the scheme have a negative net pay calculated, then they will be listed in this report. The user should then check that the pension amounts and tax codes have been entered correctly against these members before they continue with the Payroll Run.

Setting Pay Element Amounts to zero post run

If pay elements have been used for members in the payroll run, where the amount is greater than 0 (zero), then the user will be asked if they wish for these amounts to be set to zero in the Pay Element screen of Members. The following window now appears:

Pay Element Amounts to be set to zero Window

If ‘Yes’ is selected then each pay element amount used will be set to zero in the Pay Element screen of Members. If ‘No’ is selected then the pay elements amounts are not set to zero, and the program continues committing the payroll data.

Creating a BACS Data File

Members who run their payroll on Payroll Master can have their payment made by BACS. In order for BACS to be used, a data file must be created to hold the payroll data of the members.

A data file is created by many data lines. On each of these lines there must be a description of the payroll data which can be displayed on each member’s bank statement once they have been paid. This would usually be the name of the scheme who is paying the members, however, the description entered is solely down to the user. The description is held in the Bank Details screen of Schemes, and can only be up to 18 characters long.

The data file created through Payroll Master is made up from data lines. Each data line consists of the following data: Sort Code, Account Number, Account Name, Amount, Building Society Roll Number, Description.

For example, ‘64466615498325MR D M LANG 5000000000000000000000L&P SYSTEMS LTD’

Creating a Cheque Report

Members who run their payroll on Payroll Master can have their payment made by manual cheque. To inform the user of which members are to be paid by cheque, a Cheque Report will be generated during the Payroll Run, which can then be accessed through the Cheque Listing Report program.

Once the Payroll Run has been successfully completed, the following window pops up:

BACS Data File Successful Window

It is now left to the user to use the recommended WinBACS Software to transfer the BACS data file containing payroll data. Any manual cheques which need to be written out will also be left to the user.

Submit

Submit File

You will run the payroll through the same ‘Payroll Run’ program. Once the payroll run has completed you will then use the two new menu options within the ‘Payroll Run’ program called ‘Submit’ and ‘Check’.

Selecting the ‘Submit’ menu option will open up a new screen where you will enter the Scheme Code, Category Code (if applicable) and Sender. It is important that you enter the Scheme and Category exactly as you did for the payroll run itself. You will need the Accounts Office Reference number, Gateway ID and Password that has been provided by HMRC. On clicking ‘OK’ the program will generate an XML file which will then be sent to HMRC.

After the XML file has been sent to HMRC you will receive a confirmation message. This only indicates that HMRC have received the submission and not that the data is valid and has been accepted.

The Payroll Run doesn’t pick up members whose Date Finished Pension is before the Payroll Run Date, however, the Submit option in the Payroll Run does pick up extra members who have their Last Payment field ticked. When a leaver is picked up only the members figures to date will be sent along with the leaving date. Once the submission has been successful the Last Payment indicator will be unticked.

Final Submission For Year? - If this is the last payroll payment of the tax year then tick this check box.

Transferring Scheme? - This field should only be selected when you (the client) are transferring the scheme to another provider and are therefore no longer running the payroll. When this field is selected all payroll members within the scheme must have the Date Finished Pension and Last Payment fields filled in, and it’s only members with a Last Payment field ticked that are picked up.

When you know you are transferring a scheme you must do the following:

  • Run the last payroll as normal, submit and check it.
  • Fill in the Date Finished Pension and tick the Last Payment field.
  • Go to the ‘Submit’ option in Payroll Run and fill in all the details making sure the ‘Transferring Scheme?’ field is ticked. Click ‘OK’ and then ‘Check’ the submission.

N.B. It is important that the final FPS (Transferring Scheme submission) is submitted to HMRC before the new provider submits their first FPS.

Check

Check Submission

To check the status of a recent submission select the ‘Check’ menu option and enter the Scheme Code and Category Code (if applicable). Again, the Scheme and Category codes must be entered exactly you did for the payroll run itself. This will ask HMRC the status of the previous submission. You will have to wait 10 seconds, a time limit imposed by HMRC, before checking your original submission. The response will either be:

  • A list of errors in a Report Viewer window. Any errors will need to be rectified and the scheme data submitted and checked again.
  • Confirmation that the data has been accepted.
  • A message saying ‘The submission has been acknowledged by the Government Gateway’ indicates that HMRC have received your ‘Check’ request and are currently processing the data submission. You will need to check later to ensure that you receive one of the two responses above.

If you are running payroll for multiple Scheme and Scheme/Category combinations then it is important that you keep track of the stages that have been completed.

HMRC Error Messages

If a member is being paid for the first time and has no Current Pension value then a HMRC submission could return the following error:

If [STARTDATE] is completed, at least one of [STARTINGDECLARATION], [INTENDTOSTAYOVER183DAYS], [INTENDTOSTAYLESSTHAN183DAYS], [INTENDTOWORKBOTHINANDOUTOFUK] must be present or [ANNUALAMOUNTOFPENSION] must be greater than zero. Please check

A User Defined Screen has been setup called Additional HMRC RTI Data for Payroll Members in Pension Master. It currently contains 4 fields that hold additional infromation:

Starting Declaration described as [STARTINGDECLARATION] in the error message can be one of the following: A - This is my first job since 6th April, B - This is my only job and C - I have another job or pension
Int. Stay > 183 Days described as [INTENDTOSTAYOVER183DAYS] in the error message is the ‘Indicator if intention to live in UK for 183 days or more’ and should be set to ‘yes’.
Int. Stay < 183 Days described as [INTENDTOSTAYLESSTHAN183DAYS] in the error message is the ‘Indicator if intention to live in UK for less than 183 days’ and should be set to ‘yes’.
Int. Work In&Out UK described as [INTENDTOWORKBOTHINANDOUTOFUK] in the error message is the ‘Indicator if individual will be working both in / out of the UK but living abroad’ and should be set to ‘yes’.
