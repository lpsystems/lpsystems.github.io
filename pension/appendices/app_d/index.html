Introduction

This note describes the in-built support for the Euro in Pension Master.

The Euro becomes a currency in it’s own right with effect from 1/1/1999. Euro notes and coins will not be circulated until 1/1/2002 and in the transition period (which runs from 1/1/1999 to 31/12/2001) transactions may be carried out in either the Euro or the National Currency Unit (NCU) e.g. Irish Punts or Pounds Sterling.

Companies can opt to continue operating in their NCU for their businesses and accounts until 31/12/2001 or can switch to operating in Euro currency at any time between 1/1/1999 and 31/12/2001.

Base Currency

This is sometimes known as the ‘lead currency’ and denotes the currency unit in which Pension Master processes and stores information. It is anticipated that this will normally be the NCU.

Implementation of the Euro

We have implemented support for the Euro in the following ways:

System Parameters

We have introduced a new field ‘Default Base Currency’ to the first screen of this program. This holds the 3 letter code for the default base currency e.g. IEP represents Irish Punts, GBP represents Pounds Sterling. A lookup of the valid currency codes is available in this field, just click on the ‘Serach’ button and these codes are based on internationally accepted standards used in the money markets.

Schemes

We have introduced a new field ‘Base Currency’ to the first screen of this program. This holds the 3 letter code for the base currency to be used for this scheme e.g. IEP represents Irish Punts. A lookup of the valid currency codes is available in this field, just click on the lookup icon and these codes are based on internationally accepted standards used in the money markets.

This field will default to the ‘Default Base Currency’ as set out in the section ‘System Parameters’ above.

The field will initially be set to the default base currency for all schemes. As and when each scheme opts to switch to the Euro, the user will update this field to ‘EUR’ (the international symbol for the Euro). They will then be prompted that all monetary values for this scheme are to be converted from the NCU to Euro and that this process in not reversible. They will be asked whether they have an up to date backup of the current scheme data and if not, given an opportunity to exit the change and take a backup. Once this has been done, all monetary amounts for the scheme, including all membership data and historical unit prices for that scheme will be converted to Euro’s.

Values held in Annual Statutory Amounts remain in the same currency as the system default base currency as defined in System Parameters. The values should only be converted over to Euro’s at the end of the transitional period i.e. when all your schemes are obliged by law to make the Euro their base currency. Any values retrieved from the Annual Statutory Amounts table by means of the stat() formula function are automatically converted on the fly into the scheme base currency so as to ensure consistency with your scheme formulae.

The reason that the conversion to Euro process is not reversible is that when a monetary value is converted from the NCU to Euro and then back into the NCU, inevitable rounding errors will occur and thus the resulting data will not be the same as it was before it was originally converted to Euro’s.

At the end of the transition period, European law states that the NCU will become an obsolete currency, so all schemes that still have the NCU as their base currency should be converted over to Euro’s following the above procedure.

Members

All screens containing monetary values now have an additional two fields at the top of the screen which show the currency code and description of the currency that the data is held in i.e. the base currency.

The user can toggle between the NCU and Euro in these screens by using the ‘Toggle Euro’ button in Pension Master.

The above principles also apply to other areas of the scheme data that hold monetary values e.g. the scheme level user defined fields in Schemes.

Formulae

We have introduced two new functions ncu_to_eur() and eur_to_ncu() which take a monetary value and convert this from the NCU to Euro and from the Euro to NCU respectively.

It is anticipated that whilst the scheme is still using the NCU (i.e. before it has opted to switch to the Euro), all calculations will be done in the NCU and merely converted to Euro’s by the addition of extra formulae that each take the result of a benefit calculation and apply ncu_to_eur() in order to convert the value into Euro’s. Thus the result of each calculation in both the NCU and Euro will be available for output to any benefit statement or certificate.

Once the scheme has opted to switch to the Euro, and whilst still within the transition period, all monetary values and calculations within Pension Master will be in Euro’s. The Euro will now become the default currency for the scheme. The above procedure can be applied in reverse using the eur_to_ncu() function to convert calculated results from Euro’s into NCU for output to benefit statements and certificates.

Annual Statutory Amounts

This program now has an additional two fields at the top of the screen which show the currency code and description of the currency that the data is held in i.e. the base currency. The values held on this table are in the same currency as the system default base currency which is held in System Parameters.

The user can toggle between the NCU and Euro in these screens by using the ‘Toggle Euro’ button in Pension Master.

When using the stat() function in formulae,  if the scheme base currency for the scheme that the member is in is different than the system default base currency, all values returned by stat() are automatically converted into the scheme base currency and a message written to the Quotation Warnings File informing you of the conversion.

At the end of the transitional period, you should ensure that the default base currency field in System Parameters is changed to ‘EUR’ as this process also converts over the values held in the Annual Statutory Amounts table to Euro’s.
