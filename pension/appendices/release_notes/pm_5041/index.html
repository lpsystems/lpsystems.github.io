Release Notes 5.0.4b1

Details Of The Changes

  1. Amended the Part Time Service record. This makes the recording of the part time hours worked for members easier and does not now require you to enter a complete service history, regardless of wether the member was part time or not.

    If a member commences part time service then the start date, the amount of hours worked and the full time hours standard should be recorded. When the member returns to full time service or the amount of part time hours change, the end date should be entered.

    Equivalent service periods are automatically calculated in days as the user moves from each line.

    The pt(date1,date2) function is now available for use in your formulae via Formula File Maintenance. This will calculate a ratio of part time service to actual period ratio which can then be applied to service or salaries to get the part time equivalent of full time service. The manual page for this is as follows:

    pt(date1,date2)

    OVERVIEW
    Use the pt function to calculate the full time equivalent service ratio
    from a Part Time Service record.

    SYNTAX
    pt(date1,date2)

    EXPLANATION
    date1 start date for date period
    date2 end date for date period

    NOTES
    The two dates must be field references to the member file (pe_mem),
    constant dates entered in the format YYYYMMDD, where YYYY is the year, MM
    is the month and DD is the day or references to the result of another line
    of formula code within the current formula e.g. @1.

    If date1 > date2 the value returned will be 0.

    The ratio will be calculated as the total equivalent service recorded
    between date1 and date2 on the Part Time Service record plus the remaining
    service (assumed to be at full time), divided by the period between date1
    and date2.

    This is displayed as a 2 decimal place result but is available in the
    calculating formula to 8 decimal places.

    EXAMPLES
    pt(pm_des,pm_npd)
    pt(19950122,pm_dterm_dt)
    pt(19900406,19960405)
    pt(@1,19960405)

  2. Amended the cty(), cy(), cm(), cd(), hy(), hm() and pt()functions in Formula File Maintenance so that the user can now access the results of other lines of formulae as arguments to these functions in the format @line_num.
  3. For example, if you wish to calculate the period in years and completed months between the date of transfer in number 1 and Normal Pension Date, use the following formula code:
    1. transfer(ptv_date,2,1)
    2. cm(@1,pm_npd)
  4. Amended the screen selection pop-up menu in Member File Maintenance so that a window now appears that displays all the available screens for the current scheme type at once, together with the corresponding screen numbers. The user is invited to select the screen they wish to view by simply entering the screen number. This replaces the old scrolling menu that was found to be cumbersome to use. The new facility allows the user to rapidly move from one screen to another.
  5. Screen numbers are consistent regardless of the scheme type currently selected i.e. the Quotation Screen is always screen 27.
  6. Added a lookup facility in Member File Maintenance for the Query option on Status Type, Marital Status and Sex. Available by clicking on the 'Search' button.
