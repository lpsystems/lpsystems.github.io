Annual Interest Posting This Update program has been renamed to Contribution Interest Run.

Deferred Pensioner Details The existing Deferred Pensioner screen has been replaced by 3 new screens as follows:

Screen 1 - Basic Deferred Pensioner Details Screen

This contains typical deferred pensioner data that relates to the members Deferred Pension benefits. Information such as, the Date the first Payment is Due, values relating to Inland Revenue Maximum Pension/Cash and Scheme Cash, Cessation Details and the Payment Frequency. Also an additional 5 fields are available that can be defined by the user. Data must be entered on this screen before the 2 other screens can be accessed.

Note: The Payment Frequency will be set to 12 (Monthly) by default for any existing Deferred Pensioners.

Screen 2 - Deferred Pension History Screen

This is used to record any changes in the members Deferred Pension benefit. Deferred Pension Codes are used to indicate whether the values are Added/Ignored or Subtracted from the calculation of the members total Deferred Pension. Other information can also be entered such as, the Date the change is due to Commence, the Date of Last Increase and values for the members Initial/Current Pension and benefit for Spouses before and after the normal pension date.

Screen 3 - Deferred Pension Increase History Screen

This historically records any changes made to a members Deferred Pension benefit on the Deferred Pension History Screen. The last 10 alterations made to a Deferred Pension Code are displayed on the screen with options on the menu to allow any other codes to be shown. Information such as the value of the members Pension and any Spouses Benefit before and after the normal pension date are recorded.

Formulae Added a new function as follows:

rm(date1,date2,r)

OVERVIEW

Use the rm function to calculate the number of years and completed months between two dates. This is the same as the cm() function except that if the number of remaining days is greater than or equal to a specified value (given by the r argument to the function call), then the number of completed months is rounded up.

SYNTAX

rm(date1,date2,r)

EXPLANATION

  • date1 - start date for date period
  • date2 - end date for date period
  • r - is the number of days to round up
NOTES
  1. The two dates must be field references to the member file (pe_mem), constant dates entered in the format YYYYMMDD, where YYYY is the year, MM is the month and DD is the day or references to the result of another line of formula code within the current formula e.g. @1.
  2. If date1 > date2 the value returned will be 0.
  3. r must be a valid number between 1 and 31.
EXAMPLES

rm(pm_dtlft,pm_npd,20)
rm(19950122,pm_npd,1)
rm(19900112,20040523,17)
rm(@1,20040523,15)

Formulae Added a new function as follows:

if(condition,@then,@else)

OVERVIEW

Use the if function to test whether a condition is either true or false. The function will either return a constant, field name or goto a specific line depending upon the condition. If the condition is true then the first constant, field name or goto will be processed. If the condition is false then the second constant, field name or goto will be processed.

SYNTAX

if(condition,@then,@else)

EXPLANATION

condition a test condition using either a line number or field name seperated by a valid operator (either <,>,!,=). Compared with either a line number, constant (date, integer, character string or decimal) or field name.
@then either a constant (date, integer, character string or decimal), field name or a line number to goto that will be executed if the condition is true.
@else either a constant (date, integer, character string or decimal), field name or a line number to goto that will be executed if the condition is false.

NOTES

  1. Date constants must be enclosed in double quotation marks (") and be in the format YYYYMMDD.
  2. Character string constants must be enclosed in single quotation marks (').
  3. Any valid operator can be used either <,>,! or =.
  4. Any line numbers used in the condition must be previous to the if function.
  5. Any line numbers used as arguments (either @then or @else) must be after the if function.
EXAMPLES

if(pm_sex='M',@2,@2)
if(pm_scheme<@3,12,34)
if(@5!0,@5,4321)
if(@7>pm_dob,pm_rt1,32)

Formulae Formulae can now produce 3 new types of result as well as the standard decimal format. The possible types of result are as follows:
Decimal a numeric value e.g. 1234.56
Date a valid date e.g. 01/12/1999
Integer a numeric value with no decimal places e.g. 1234
Character a string up to 100 alphabetic or numeric characters e.g. Mr G. Marx
The type of result will depend upon the first section of the formula line.

For example, if 'Pensioner' is the first section of the line then the line is set as a character line. Also if #pm_surname is the first part of the line, or any function returns a character value then the line is a character line. This means that numeric operators cannot be used on the line or the result of this line. e.g.

1 #pm_surname
2 @1 * 5
3 result @2
When this formula is run it would produce a warning on the Warnings Report saying that an attempt was made to perform a calculation on a character field.

Calculations can still be performed on Dates e.g.

1 #pm_dob + 5
2 result @1
This will return a date that is 5 calendar days after the members date of birth.

Formulae ir_norm_revmax(pp,date) OVERVIEW

Use this function to calculate a revenue maximum fraction under Irish Revenue Rules for a normal retiree.

SYNTAX

ir_norm_revmax(pp,date)

EXPLANATION

  • pp - denotes the type of Revenue Maximum. The valid Parameters are as follows:
    • NR - Calculate fraction for Normal Retirement Pension
    • NL - Calculate fraction for Normal Retirement Lump Sum
  • date - denotes the date to project the revenue maximum fraction to. This can be either:
    • @line_num i.e. a reference to a previous line of your formula that evaluates to a date
    • pm_npd field on pe_mem.
    • a fixed date if the format YYYYMMDD e.g. 20000201 for 1st February 2000.
NOTES
  1. pp must be in uppercase.
  2. ir_norm_revmax uses Date Entered Company (pm_dec) (to be found on the first screen of the Member File). If this date is blank for a member then a value of zero will be returned and a message written to the warnings report.
  3. If the date returned by the second argument to the function is blank then a value of zero will be returned and a message written to the warnings report.
  4. The function uses the value evaultaed by the second argument to denote the date of normal retirement, which could be used to quote maximum benefits at age 60 for a scheme that allows retirement at any age between 60 and 65, for example.
  5. Full account of any part time service (as defined on the Part Time Service record in Members) is taken account of. If a member has been in continuous part time service, they are treated as if they were in full time service in the calculation of the revenue maximum fraction as per Revenue Rules.
  6. ir_norm_revmax generates the Revenue Maximum fraction and NOT the value of a Revenue Maximum pension or lump sum. To get the Revenue Maximum pension or lump sum you will need to multiply the result of the call to ir_norm_revmax by a suitable salary.
EXAMPLES

ir_norm_revmax(NR,@1)
ir_norm_revmax(NL,20000201)
ir_norm_revmax(NL,pm_npd)

Formulae pen_act_proj(reval_rate,date) OVERVIEW

Use the pen_act_proj function to calculate the projected fraction required under the Irish Pensions Act for leavers from a Final Salary pension scheme with deferred benefits. This fraction is used to calculate the members pension on leaving the scheme, but revalued to normal pension age.

SYNTAX

pen_act_proj(reval_rate,date)

EXPLANATION

  • reval_rate - is the rate of revaluation to be assumed for projection purposes. Legislation dictates this is 4%p.a. or the increase in the CPI if less. Enter any rate you require here, including rates with a decimal point e.g. 2.5%. reval_rate can also be the result of another line in your formula in the form @line_num e.g. @2.
  • date - date pensionable service commences. This can be either:
    • @line_num i.e. a reference to a previous line of your formula that evaluates to a date
    • pm_dec, pm_djla, pm_djp, pm_dps or pm_dco field on pe_mem.
    • a fixed date if the format YYYYMMDD e.g. 20000201 for 1st February 2000.
NOTES
  1. Full account of any qualifying service in respect of any transfer in that the member may have is taken account of.
  2. Any non-reckonable service for the member is taken account of. Non reckonable service is taken as one or more lines on the part time history screen where the part time hours field is 0 but the corresponding full time hours field is > 0.
  3. The amount of the revaluation is applied between the date of leaving the scheme (as calculated in 1. above) (subject to a minimum of 1/1/96) and the 1st January immediately prior to normal pension date, and is calculated to years and completed months.
  4. The revaluation rate used for the projection will be that supplied by the reval_rate argument to the function.
  5. Date of leaving the scheme is calculated as follows:
    • If Date Left (on first screen of Member File) is available use this.
    • otherwise, use Quotation Termination Date.
EXAMPLES

pen_act_proj(3,20000201)

Historical Status Report This report, which can be found on the Membership Analysis sub-menu, produces a report of members status codes at a specified date in the past for a selected group of members. It replaces the old Member Status Analysis Report as it provides the user with a greater selection criteria for the members they wish to select, and it allows the user to enter in an effective date which determines from what point in time the report is to be produced from. You have the option of doing the report in either Summary or Detailed format.

The Summary Report will provide you with the number of members held under each new status code using the selection criteria and the effective date entered by the user.

The Detailed Report will provide you with details of each individual member held under each new status code, these including the member's name and number, the effective date, old and new status codes, and the date when the status code was updated. Load Renewal Data This module can be found on the Updates sub-menu. It allows the user to load up renewal data from a file into the Batch Data Input module of Pension Master. By entering in the required details through the selection criteria, the program will firstly check that the details are correctly entered. Secondly, the fields which make up the data lines within the data file will be checked for the correct format.

If all the data lines held within the data file are correct then they are loaded into the database. However, if the data is found to be incorrect then a report is produced informing the user where the errors are in the file.

The program has one main objective which is to load up renewal data provided by the user via a data file. It is therefore expected, by the program, that the data file being used is in the required format. In order for the user to ensure this, they should use a software package, like Microsoft Excel, to reformat the data file prior to using this program.

You will need to make this extra menu option available to users as appropriate by granting access using the Users module of Pension Master. By default all new menus and menu options added to Pension Master will not be switched on for any users.
Load Renewal Data In the GUI version of Pension Master, upon entering the data filename field, a pop-up window is displayed allowing the user to choose the data file to load from their local PC e.g. they can load the data file directly from a floppy disk in the floppy disk drive on their PC.
Load Renewal Data There has been a pop-up window added to the Load Renewal Data program. The window gets displayed to the screen if a data line has been fed through the program which contains an indicator that belongs to two or more members, i.e. two members have the same NI Number.

The members who are allocated to this indicator are listed in the window and the user is then able to select which member the data line belongs to. If the user does not select a member from the list by clicking on 'Cancel', then an error message is sent to the exceptions report as it is un-known which member the data line is intended for.

Load Renewal Data There have been two new options added to the renewal type in the Load Renewal Data program. As well as the original three options of Contributions, NI Contributions and Salaries, there is now also Marital Status and Company Code. These two new options will allow for the update of the respective member detail of marital status or company code. The data file for these options will take the format of containing two fields: Identifier and either Marital Status or Company Code, depending on the renewal type being used.

The data file will go through the same checking procedure as the original three i.e. checking that the correct format is used, checking that the data line contains the correct number of columns etc., any errors found will be sent to the exceptions report. Once the loading is complete there is a note sent to the audit trail for each data line loaded, stating that it was the Load Renewal Data program that made the updates. The alterations can be viewed through the Members.

If the data has been loaded with data type of Company Code, then the company history record will also be respectively updated. The Effective Date and the Date Updated held within the company history record will be set as today's date.

N.B. The company history record will only be updated if the new company code is not the same as the original company code i.e. If a data line is loaded with company code of '0000' to update a company code of '0000' then no record of this update will be sent to the company history record, or to the audit trail.

Pension Increase Run The Pension Increase Run has been changed to integrate with the new Pensioner and Deferred Pensioner screens in Members.

New pension values can now be calculated based upon either a percentage rate or the result of a formula. Care must be taken to ensure that the formula generates only the increase for the period. An extra option has been included to allow increases to be calculated on a pro rata basis.

The program is split into 4 main sections described as follows:

Provisional This allows an increase run to be performed but does not update the Pension/Deferred Pension History.
Actual This allows an increase run to be performed and does update the Pension/Deferred Pension History.
Undo This allows an increase run to be removed from the Pension/Deferred Pension Increase History screen and reverts the values on the History screen back to before the Actual Increase run was performed.
Delete This allows a previously saved Increase Parameter to be removed.

The members that are increased can be selected using standard selection criteria (i.e. Scheme, Company, Category, Status, Special Code and Member). From the members selected a list of the Pension Codes currently used is displayed allowing either all the codes to be updated with a global Formula/Fixed Rate or to be increased individually.

The information used to select the members and the specific increase information used against each Pension Code can be saved as an individual set of parameters (known as a Pension Increase parameter Set). These parameters are specific to each scheme and each pension type, for either Deferred Pension or Pension increases. This allows them to be saved, using a descriptive name, for future use.

A report is generated if exceptions are found during the Provisional or Actual Increase Run. This shows the members details, the options that have been selected and the reason why the update was not processed. An increase report is created showing the members pension values, that have been correctly calculated, before and after the Pension Increase Run.

See the Pension Increase Run page for further details.

Formulae unalloc(fld,nn) OVERVIEW

Use the unalloc function to calculate unallocated contributions for a specific investment fund in Contribution History.

SYNTAX

unalloc(fld,nn)

EXPLANATION

  • fld - is the name of a field on the Contribution History file (pe_defhst_b).
  • nn - is the investment fund number required. If investment fund 00 is used, this function will total all units for the field specified across ALL investment funds the member invests in.
NOTES
  1. nn must be between 00 and 20.
EXAMPLES

unalloc(pdb_ee_ord_c,01)
