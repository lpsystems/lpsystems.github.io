Release Notes 5.0.6

Details Of The Changes

  1. Extended the security levels available for the system administrator to use from 1-3 to 1-99. This gives the system administrator much greater flexibility in controlling user access to member and scheme records in Pension Master.
  2. Amended the development of Company, Category & Status Histories. When adding a new member, the user will be prompted for an effective date for Company, Category and Status when leaving these fields (Category and Status effetive dates will default to the date entered for Company). This allows a history to be built up without being forced to use Date Entered Scheme as a starting base. If these fields, are updated a similiar effective date prompt will appear and will require an entry.
  3. Amended the Menu selection in Scheme File Maintenance and Category File Maintenance. When a user selects the menu option, available screens will be displayed in a similar way that the member record screens are displayed in Member File Maintenance. In the case of Category File Maintenance, existing Quotation Types will be available for direct selection.
  4. Fixed problem in Mail Merge Integration Report where the Date Comparison field was being ignored. This only manifestied itself if you ran the report directly rather than using the Letter/Cert option in Member File Maintenance. Updated the help text and the User Manual to reflect this.
  5. Extended MLI Factor Maintenance to allow the updating of RPI factors. The RPI factors can be updated in exactly the same way as the MLI's. The inclusion of these factors will allow salary dynamisation in formulae (see later point). The menu option will change to MLI/RPI Factor Maintenance.
  6. Added a new Scheme User Defined Details screen to Scheme File Maintenance to allow users to define and hold up to twenty extra fields at a scheme level. The values of these fields are validated against a field format in the same way as the User Defined Screens on the Member File. The help text for this new screen is as follows: Scheme User Defined Details This screen will allow users to define and hold up to twenty extra fields at a scheme level. The selected fields can be tied to a specific format type in order to validate input. The valid types include Date (DT), Indicator (IN), Numeric Integer (NI),  Numeric Decimal (ND) and Free Text (TX). The user can spread the fields across the record by leaving description  and format fields blank but a format cannot be entered unless a  description exists.
  7. Removed warning message from Member File Maintenance when selecting the Transfer in or Transfer out screens if their is no data present.
  8. Added a List sub-option to the Report menu option in Factor File Maintenance. This allows you to print off a list of the factor tables currently on Pension Master.
  9. Revised the Reports from the Scheme File Maintenance. The Details report has been reordered to follow the menu selection order and irrelevant screens will not be printed e.g. Units Prices will only be printed for a Defined Contribution or Hybrid scheme. The List report now shows participating companies and categories separately with respective totals.
