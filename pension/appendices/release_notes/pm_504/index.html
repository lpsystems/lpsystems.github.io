Release Notes 5.0.4

Details Of The Changes

  1. Added new Part Time Service record. This will allow users to record the part time hours worked for members.

    The record relies on a complete history being recorded i.e. if the member joined as full time, then the record should show the period as full time e.g. 40 hours worked out of 40.

    When the members hours change then the period should be closed off at the appropriate date and a new line opened at the new period start date with the new part time and full time hours.

    Equivalent service periods are automatically calculated as the user moves from each line and total equivalent service is calculated when the input is accepted.

    The next system release will allow the user to incorporate this record in formulae and reports. This record is being released now to enable users to record part time hours details in a proper format prior to the full facility being available.

  2. Changed the way Pension Master calculates expenses when taking expenses off incoming contributions. This is no longer done at the Investment Manager level as it was before. Instead, a single set of expenses applies across all investment funds within a scheme. We have also replaced the Fund % Expense with a Salary % Expense. This allows you to take a percentage of salary (you define the salary type you wish to use - Gross, Scheme, Pensionable, Miscellaneous 1 or Miscellaneous 2) as an expense. See the User Manual for further details.
  3. Added extra display fields Release Date and Licensed To to the splash screen which appears upon starting Pension Master. This should allow L&P Systems Limited support staff to more effectively track the versions of Pension Master installed at client sites.
  4. Changed the main menu in the Unit Posting program as follows:
    • Changed the text of the Proceed menu option to Post_units.
    • Removed the old uNpost menu option. The Undo menu option now combines the capabilities of the old Undo and uNpost menu options thus removing the confusion some users were having over the difference of these two menu options. The new combined menu option can be used for:
      • Unposting units for a user specified group of members and a given contribution date.
      • Re-setting the unit posted flag on a particular contribution date to "N" for a user specified group of members.
  5. Changed the Update Conts/Units flag on the Scheme File Maintenance program to read Allow Manual Update Of Units?. This flag is now has the following implications:
    • It determines whether a user can update the unit history of a member of the scheme by directly editing units on that members Unit History screen in Member File Maintenance. We would generally recommend that you do not directly edit a members unit history screen other that for manually loading cumulative units to date (for example where you are taking over the administration of a scheme and are setting up a starting point for future unit allocation for the scheme). To adjust a members unit history we would recommend that you use the Undo menu option in the Unit Posting program to reverse the unit allocation for that member and then make any adjustments to the contribution and unit price entries for that date before re-posting that contribution to units. This ensures a proper reconciliation of the whole process within Pension Master and avoids any errors that may have otherwise occurred using manual editing of units.
    • It determines how the unit posting flag on new contributions loaded onto Pension Master is set. If the flag on the scheme file is set to "Y", then all new contributions loaded onto Pension Master (whether through Batch File Maintenance or Member File Maintenance) will automatically have a unit posting flag of "Y" allocated to them. Conversely, if the flag on the scheme file is set to "N" then all new contributions will be loaded with a unit posting flag of "N". The exception to this rule is for defined benefits or final salary type schemes (where the Defined Contribution Scheme flag in Scheme File Maintenance is set to "N"). In this case, all new contributions loaded onto Pension Master will automatically have a unit posting flag of "Y".
    • We would generally recommend that the Allow Manual Update Of Units? flag is set to "N" for all schemes unless you specifically wish to directly edit a members Unit History. In this case, set the flag on the Scheme File to "Y" whilst you make the edit to the Unit History and then immediately re-set it to "N".
    • If you add a contribution onto Pension Master whilst the Allow Manual Update Of Units? flag on the scheme file is set to "Y" and wish that contribution entry to be flagged as unposted (i.e. you wish to use this contribution to allocate units) you should use the Undo menu option in the Unit Posting program to re-set the unit posting flag for this contribution entry for each member concerned for that contribution date.
    • If you try to add contributions onto Pension Master whilst the Allow Manual Update Of Units? flag on the scheme file is set to "Y" and the current scheme is either defined contributions or hybrid, either through the Contribution History screen in Member File Maintenance or via the Batch Data Input program, a warning message pops up as follows: Update Units Flag = "Y". Any New Conts Will Default The Unit Post Flag To "Y". You are then asked if you wish to proceed and enter contributions or return to the main menu. This is intended to indicate to the user that they will not be able to post these contributions to units without first using the Undo menu option in the Unit Posting program and that they might consider setting the Allow Manual Update Of Units? Flag on the scheme file to "N" before entering the contribution details.
  6. The unit posting flag on each contribution entry is now displayed on each members Contribution History screen in Member File Maintenance. This is shown in the last column on the Contribution History screen.
  7. Users can now only edit contribution entries on the members Contribution History screen in Member File Maintenance where:
  • The contribution entry is not a Switch or a Transfer i.e. "S" or "T" does not appear in the S/T column on the screen.
  • The contribution entry has a unit posting flag of "N" i.e. contributions that have already been used to buy units cannot be directly edited without unposting the relevant units first (using the Undo menu option in the Unit Posting program).
