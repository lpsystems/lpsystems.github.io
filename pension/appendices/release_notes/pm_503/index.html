Release Notes 5.03b5

Details Of The Changes

  1. In the Salary History, Contribution History and Units History screens of Member File Maintenance, check that the history date entered is not less than the date the member joined the scheme (pm_dps)., If it is, display an error message and ask the user if they really intend to use this date.
  2. Added postcode field to Company File Maintenance and amended Letter/Cert option to reflect this.
  3. Added Status Code to the list of possible membership selection criteria in Pensionable Salary Update program.
  4. Added Status Type and Special Code to the list of possible selection criteria when searching for a member record in Member File Maintenance. Also added an F8 lookup facility on the Special Code field.
  5. Added Calculation Trace Report to both Member File Maintenance and Batch Quotation Run so that every time you run a calculation on Pension Master, a report is produced showing for each member and each formula, the formula lines processed together with intermediate results for each formula line.
  6. Extended Label Printing Facility in Member File Maintenance to give more flexibility in output. First of all, the user can choose to print on to a single strip of labels, as before, or on to a laser sheet (2*4) similar to an Avery L7165 sheet. The user is no longer restricted to just producing labels showing the members name and address in company order. A free text header line is available followed by six lines of selected data fields (eight line with the laser sheet option) from the Members Basic and Address records and Scheme, Company and Category descriptions. The labels are automatically sorted in alpha order but the user can extend this sort using scheme code, company code and/or category code. All selections have a F8 lookup facility and basic help text attached.
  7. Added report code, description and scheme code and description to the header of the Mail-Merge Variable Definition Report produced by the Mail-Merge Integration Report at the request of Mark Winstanley.
  8. Allow use of F2 key to delete lines of contribution details in the Contribution Hiostory screen of Member File Maintenance. This is allowable in the following circumstances:
    • If the scheme is a defined benefits scheme.
    • Otherwise, if the unitposting flag for that contribution entry is set to "N" i.e. if that contribution has not yet been used to purchase units.
  9. Fixed bug in Quotation Screen of Member File Maintenance where current and previous quote dates were not being written back to the database prior to the calculations being run, possibly resulting in last time's quotation dates being used in any calculations that depend on quote dates. This is not expected to affect many users as most formulae depend on termination date and not quote date.
  10. Display batch date, scheme code and scheme name at the top of the second screen in Batch Data Input so as to make it clearer to the user which batch they are inputting.
  11. In the Salary section of Batch Data Input, the pop-up deducible window for Pensionable Salary now appears not only when the Pensionable Salary is blank, but also when it is zero. This is to aid input of salaries when returning to a previosly saved salary batch in the situation where you have not completed entering the salaries prior to saving the batch.
  12. Added prinouts of investment funds > 0 to the report option when inputting a Contribution Batch in Batch Data Input. This enables the user to check things like expenses and allocations to individual funds that were not previously available at this stage.
  13. Contributions screen report in Member File Maintenance now prints details of the breakdown of any expenses for each contribution source and defaults to printing in landscape mode.
  14. Added Contribution History and Unit History to the Full Member Report in Member File Maintenance.
  15. User can now select date fields pds_date, pdb_date and ptv_date using the defsal(), defhst() and transfer()functions in Formula File Maintenance.
