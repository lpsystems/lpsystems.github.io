Release Notes 5.0.8

Member File Maintenance

Added a new field pm_age to Member File Maintenance to store age for the member. This is also displayed on the Basic Member Details screen. The field is dynamically calculated each time you go into this screen, add a member, run a Member Report or Mail Merge Integration Report or a formula that references this new field. The way this is calculated differs on which program you are using as follows:

  • Member File Maintenance - always calculate age at today's date i.e. the system date.
  • Member Report/Mail-Merge Integration Report - always calculate age at the report date.
  • Formula File/Quotations - always calculate age at quotation date.

System Administrators: The field will default to calculating age last birthday.  If you wish your Pension Master system to calculate age next birthday instead, and to retain that for all future use, you will need to change the Calculate Age Next (N) or Last (L) Birthday field in the System Parameter Maintenance program as appropriate.

Category File Maintenance

Added new entry validation fields for Maximum Age For Joining. These fields are set up for Full and DIS Only Membership and will default to the highest of the male and female NPA if left blank.

Date Joined For LA and Date Joined For Pension fields on the Member Basic Record will be validated against these maximum fields and users will be warned if the dates entered exceed them.

Member File Maintenance

Added a new flag Capitalise Names/Text Fields (Y/N) to the System Parameter Maintenance program to allow the user to switch on the automatic casing of certain fields. At present the following fields are supported:

  • Title
  • Forename
  • Surname
  • Job Title
  • Member's address
  • Member's salutation
  • Full forenames

System Administrators: The field will default to casing these fields. If you wish your Pension Master system not to do this, for example, if you are an Irish site with Gaelic surnames, you will have to change the Capitalise Names/Text Fields Y(/N) field in the System Parameter Maintenance program to 'N'.

Member File Maintenance

Extended the member search facility so that when more than one member is selected using the Query option, a search list is displayed to the screen showing basic details of each member selected. This allows the user to navigate the list and once they have found the member they require, press Return to select the record. The total number of member records selected is also displayed to this list for information purposes only.

This replaces the old member selection method whereby the first record of a group of members was displayed to the screen and the user then had to use the Next and Previous keys to navigate between the members.

Once a member record has been selected using the new method outlined above, the old Next and Previous keys are still available should the user wish to scroll through the records in the old way.

For users of the new GUI release of Pension Master, a Quick Search button is available which will re-display the pop-up window showing basic details of all the members selected to enable the user to quickly locate any other member records they wish to update/view.

Details of how these fields are implemented in Pension Master are to be found in the Security Features In Pension Master chapter of the Pension Master User Manual.

Member File Maintenance

Added validation for the NI number field so as to force it to conform with rules set down by the Inland Revenue/Contributions Agency.

If the user does not know a member's NI number and they have input a valid date of birth and sex for the member, upon entering the member's date of birth, the user is asked whether they wish Pension Master to generate a temporary NI number for the member. If they do, a temporary NI number is generated in the format TNDDMMYYX where DD = day of birth, MM = month of birth, YY = year of birth and X = 'M' or 'F' dependent on the member's sex, and displayed to the NI number field on the screen.

Member File Maintenance

Amended the Company, Category and Status History screens to allow users to add lines of history and amend old and new codes. The system will carry out various checks to ensure that the Old Code matches the New Code from the previous line and also checks that the entered codes are valid. Effective Dates are also validated to ensure that histories are entered in chronological order.

A link between the New Code of the latest entry on each of the history screens to the members Company, Category and Status fields shown in the top box of each member record screen. If the latest New Code differs from the members current value for Company, Category or Status then they will be warned that the current value will be updated if the screen is saved followed by a prompt to allow the user to prevent this.

Reports

One new report option has been added in the Standard reports sub menu:

  • The Temporary/Missing NI Numbers Report can be used to check NI numbers that have not been entered, or temporary NI numbers which are incorrect. The report suggests a temporary NI number based on the rules out-lined in the Member File Maintenance section above. If the Date of Birth or Sex have not been entered then a temporary NI number cannot be generated. In this case a comment is printed showing the action to be taken. If correct temporary NI numbers can be generated, then the user will be asked if an automatic update is required.

Three new report options has been added in the Membership Analysis reports sub menu:

  • The Member Status Analysis Report allows the user to produce a count of the Members per Scheme grouped by their current Status code. The report will be of great use following the take on of new scheme data and will aid the checking of members being loaded.
  • The Unposted Contributions Report can be used to check units that have not been posted to Investment Funds. The report can be generated to show Member and Unposted Contribution details with sub-totals for each member. The other format is the total Unposted Contribution for each Investment Fund with totals by Scheme.
  • The Unposted Switch/Transfer Report allows the user to check members that have incomplete Switch/Transfers remaining. The user can select members by Scheme or report on all member records in Pension Master.

One new report option has been added under the Miscellaneous Reports sub menu:

  • The Reconciliation Report can be used to check on the number of members in a scheme or selection of schemes and also their total salaries, contributions and the number of lines of data in these two records. The report can be run to give member totals or just scheme totals depending on the option selected. We envisage this report being of great use following the take on of new scheme data and will aid the checking of the loading of members and salaries and contributions.

A new suite of report options have been added under the Standard Reports sub menu called Current Actives, Current Preserveds and Current Pensioners Reports. These reports have been created to provide a standard set of basic details for those selected members whose status code has the appropriate status type (A, D or P).

System Administrators: You will need to make these new menu options available to users as appropriate by granting access using the User File Maintenance module of Pension Master. By default all new menus and menu options added to Pension Master will not be switched on for any users.

Member File Maintenance

Swapped positions of Date Joined For Life Assurance and Date Joined For Pension fields round on the Basic Member Details screen in Member File Maintenance so as to be more meaningful to the user. The fields are now displayed with Date Joined For Life Assurance date before Date Joined For Pension.

Member File Maintenance

Amended the way Pension Master creates a contribution entry in respect of a transfer in for defined contribution or hybrid type schemes. This only applies for Irish schemes. UK schemes continue to be handled in the way they have been for some time, which is set out in the User Manual. The contribution entry is calculated as follows:

    EE ORD = MEMBERS ORD CONTS INCLUDED
    EE AVC = AVC'S INCLUDED

    ER ORD = TRANSFER VALUE - (EE ORD + EE AVC)

    EE PRRTS = 0

    ER PRRTS = 0

Member File Maintenance

After running a quotation, the quotation screen print now includes a printout of any member notes (if they exist) and the basic member indicator details from the Member Indicators screen. It is envisaged that these will be used as a useful tool when independently checking calculations prior to quoting them to a member.
