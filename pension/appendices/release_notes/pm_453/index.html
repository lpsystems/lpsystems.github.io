Release Notes 4.5.3

New Features

  1. Fixed bug in ir_revmax() in Formula File Maintenance in the case of the use of the EL/DL option. The bug arose where a member's actual service was less than 20 years. This is detailed in note 10 of Joan Whelan's note of 18/12/96.
  2. Fixed bug in Batch File Maintenance when adding a member to a contribution batch where the scheme concerned is defined as a hybrid scheme in Scheme File Maintenance. The system now correctly checks that the member has an investment choice set up on his/her record in the Member Investment Choice screen of Member File Maintenance. If not, the program prompts the user to do so before allowing them to add any contribution details for this member to the batch. This is covered by note 9 in Joan Whelan's note of 18/12/96.
  3. The avgmin() function has been removed from Formula File Maintenance as this was superfluous to requirements and has never been used by anyone despite being available in Pension Master for a number of years now.
  4. Fixed a small bug in the expense calculation routine for cases where contributions were added/updated through the Member File Maintenance program. This was causing expenses to be added on top of existing expenses rather than be re-calculated from scratch every time a contribution was updated. This is detailed in note 3 of Joan Whelan's note of 18/12/96.
  5. Following on from point 4) above, expenses for a particular contribution entry are now calculated as the cursor leaves the row on the screen for that particular contribution entry and not as it used to be where expenses were only calculated when the user saved the contribution details by pressing Esc at the end of editing contribution details.
  6. Furthermore, expenses are only re-calculated for an existing contribution entry if the user actally changes some of the contribution details for that entry e.g. alters the amount of the employee ordinary contributions.
  7. Altered the way errors are handled by the Pension Master software so that when an error is encountered a window opens up on the screen with a red background and displays the error message. The user is then required to press any key to continue. This contrasts with the old sytle error messages which were displayed in red reverse on the line at the bottom of the screen and were often unnoticed by users.
  8. Added an extra field entitled Document Name to the first screen in the Mail-Merge Integration Report. This can be used to store the name (and directory if required) of the document you wish to mail-merge your data into e.g. G:\JOHN\TEST.DOC.
  9. Removed the Letter/Cert option from the Scheme File Maintenance main menu as this is only relevant to those users who have Uniplex as their default word processor and was never really used much anyway.
  10. Amended the Mail-Merge Integration Report for users of Microsoft Word so that immediately after the report is processed, it calls a shell script called merge.com which can be used, for example, to move the merge data document merge.doc to another directory which may be of use to those instances where Pension Master is setup to co-exist with a Novell Netware network. Simply edit the file merge.com which is in the /u/LandP/progs directory to perform the necessary actions.
  11. Fixed a bug in avgmax() where if there were three or more salaries in the same calendar year, the highest of the three was not necessarily used in the calculations.
  12. Added a new sub-menu OUTPUT to the Member Report program which allows the user to choose whether they wish to produce the report in normal report format or in comma-separated value (.CSV) format. The .CSV format is suitable for importing data directly into numerous DOS/Windows applications e.g. Microsoft Excel. See the separate note produced by L&P Systems Limited which gives further details of how to do this.
  13. Amended the Member Report in the case where the user chooses the eXport menu option so that immediately after the report is processed, it calls a shell script called pension.com which can be used, for example, to move the data document pension.csv to another directory which may be of use to those instances where Pension Master is setup to co-exist with a Novell Netware network. Simply edit the file pension.com which is in the /u/LandP/progs directory to perform the necessary actions.
  14. In the Unit Posting program, only ask the user to print the report off if some units were actually posted.
