Release Notes 7.0

Audit Trail Listing A 'Clear' menu option has been added to the Audit Trail Listing. The option opens a window showing the oldest Audit Trail record and allows you to enter a date so that a range of audit records can be reported on and then deleted. This allows you to keep, for example, a years audit trail records by removing the oldest month on a monthly basis.
Batch Data Input A new facility has been added to the Post option. It allows the user to state what should happen to the data when it gets posted. The options are as follows:

Add - Add the value to the entry that already exists for the member for the batch date. If there is no data there for the batch date, then insert the new data.
Insert - Only save data on member record for the batch date, if data doesn't already exist. If data does exist, then ignore it.
Ignore - Do not make any updates at all for this.
Replace - Replace any data that exists for the specified field for the batch date on the member record. If there is no data on the member record, then insert the new data.

Batch Data Input A new menu option has been added to the menu called 'Undo'. It allows the user to remove data for posted batches already held in this program.
Batch Data Input When using the Post option, you can now either do a Provisional or an Actual run.
Categories When setting up a quotation in categories, it is now possible to allocate runtime-input screens to be used with the quotation. Clicking the "Runtime" button will bring up a list of those available. These will need to have been set-up elsewhere first.
Categories Two new fields have been added to Contribution & Interest Details. Formula Code allows a valid formula to generate an interest percentage. The Pro Rata field works with either the Formula Code or Interest fields to calculate interest for partial years. This uses the last interest run date (screen 3 on Contribution & Interest Details in Members) and the Interest Date that is entered when running a Contribution Interest Run.
Categories NEW SCREEN. UDS/UDA Allocation screen has been created to allow the user to allocate UDS and UDA screens, set up at scheme level, to the categories within that scheme. This will be useful if the UDS/UDA screens have been created with the purpose of only members of a certain scheme being able to access them.
Contribution Interest Run A run report will now be produced listing all the results, which are generated from the Contribution Interest Run. The members in the selection, are updated with these results.
Current Members NEW program. This program has been created to allow the user to generate a report, listing all members currently held on Pension Master, based on a selection criteria entered by the user.
Diary When logging into Pension Master, only show the diary file if there are some items in the diary for the current user with a date <= today's date i.e. there are some outstanding diary items the user needs to address.
Euro Currency Conversion A new option has been added to the System Data menu, this is Euro Currency Conversion. This screen can be used to hold details of the value of any currency against the euro as at a given date. It is also possible to record whether or not the exchange rate gainst the euro has been fixed. Any code given to a currency on this screen can also be used in the euro_rate(currency_code) formula.
Factors Historical Factor records can now be held on Pension Master. By choosing to update a Factor record, and changing the date, a historical record is created. Any formulae lines that reference term related factor tables have been changed from :00001(@1) to :00001(@1,TODAY). The same argument default applies to tables referenced using ::.

invtot(cont_type,func_type,fund,type,flag,date1,date2)

This function has replaced defhst, defhst_t, unit, unittot, conttot and exptot functions from version 6.2 of Pension Master.

OVERVIEW Use the invtot function to calculate the total investments between two given dates for a particular source.

SYNTAX

invtot(cont_type,func_type,fund,type,flag,date1,date2)

EXPLANATION

  • cont_type - is the contribution type code
  • func_type - denotes whether you wish to do a total for contributions (C), expenses (E) or units (U). Note the capitalisation.
  • fund - is the investment fund code
  • type - denotes the transaction type you wish to include e.g. BUY, TRANSFER, SWITCH. This should default to BUY.
  • flag - denotes which date on pe_minv table to do the comparisons on as follows:
    • 1 - date due
    • 2 - date received
    • 3 - date invested
  • date1 - is the start date for the period required
  • date2 - is the end date for the period required
NOTES
  1. If date1 > date2 the value the total will be calculated with a term of 0.
  2. The two dates must be field references to date fields on the member file (pe_mem), constant dates entered in the format YYYYMMDD where YYYY is the year, MM is the month and DD is the day or references to the result of another line of formula code within the current formula e.g. @1.
  3. Fund can be set to 'TOT', this means look at the total contribution rather than that part of the contribution for an individual fund in the case where a member invests their contribution in more than one fund.
  4. If func_type is set to 'E' then fund MUST EQUAL 'TOT'.
  5. If func_type is set to 'U' then the fund CANNOT EQUAL 'TOT'.
  6. If fund equals 'TOT' then FLAG CANNOT EQUAL 3.
  7. If func_type is set to 'U' then the type CAN EQUAL 'ALL' which will allow all transaction types to be included in the calculation.
EXAMPLES invtot(ERORD,C,EQUITY,BUY,1,19950112,pm_npd)
invtot(EEORD,E,TOT,BUY,1,pm_dps,pm_dtlft)
invtot(EEAVC,U,MANAGED,TRANSFER,2,19950112,20031115)
invtot(EEAVC,U,MANAGED,ALL,1,19950112,20031115)

Formulae

loop(line_ref,max_loop)

OVERVIEW Use the loop function to allow the user to loop back to a previous line in the formulae.

SYNTAX

loop(line_ref,max_loop)

EXPLANATION

line_ref - to be entered in the format of @n, where n is the line number e.g. @1.
max_loop - this represents the maximum number of times the formula should go through the loop. This should be either a number or a reference to a previous line e.g. 20, @1.

NOTES

If max_loop is entered in the format @n, then the value held on the referenced line must be numeric.

EXAMPLES

loop(@2,15)
loop(@4,@2)

Formulae

fld => 'str' goto @ln

OVERVIEW

Use the => operators to compare a field value with a character string as part of an 'if' type branching process.

SYNTAX

fld => 65 goto @ln

NOTES

  1. If the comparison is a date string then the " (double apostrophe) must be used to surround the string.
  2. If the comparison is a character string then the ' (single apostrophe) must be used to surround the string.
  3. If the comparison is a numeric/decimal value then no apostrophe is needed.
EXAMPLES

#pm_dob => "19990101" goto @5
#pm_age => 65 goto @5

Formulae

saltot

OVERVIEW

Use the saltot function to calculate the total salaries between two given dates for a particular salary screen and salary column.

SYNTAX

saltot(sal_scrn,col_num,date1,date2)

NOTES

  1. The sal_scrn must either be a numeric value between 1 and 4, or it must reference the result of another line of formula code within the current formula e.g. @1.
  2. The col_num must either be a numeric value between 1 and 5, or it must reference the result of another line of formula code within the current formula e.g. @1.
  3. If date1 > date2 the value the total will be calculated with a term of 0.
  4. The two dates must be field references to date fields on the member file (pe_mem), constant dates entered in the format YYYYMMDD where YYYY is the year, MM is the month and DD is the day or references to the result of another line of formula code within the current formula e.g. @1.
EXAMPLES

saltot(1,2,19900112,pm_npd)
saltot(@1,4,pm_dps,pm_dtlft)
saltot(2,1,@2,20031115)
saltot(4,5,19950112,@3)

Formulae

special(group,code)

OVERVIEW

If the member already has the Special Code allocated to them, then they will be placed in the Special Code Group.

NOTES

  1. The Special Code Group can be any 8 character code and is not validated.
  2. The Special Code must be a valid code and must be allocated to the member scheme.
  3. If the Special Code Group and Special Code have already been given to the member then nothing is done and no warning is produced.
  4. The function returns no value.

General Data Load NEW program. This replaces the Load Renewal Data program. It will provide the user with the ability to carry out the same 5 loads into the following database tables: Salary Batch Details
Contribution History Batch Details
N.I. History Batch Details
Basic Member Details (Marital Status and Company Code loads)

The program has the potential to load into more database tables. This will become available in future releases.

Import Translation NEW program. This has been created to be used in conjunction with Rule 7 of the General Data Load program. The user can set up a translation code which will hold data found in an import data file and Pension Master's data equivalent. When this code is then used in Rule 7, it will translate the import data into Pension Master data.
Investment Reconciliation NEW program. This program produces a report out to Microsoft Excel of investment details, as selected by the user, for a given group of members, for either all investment funds or just one selected fund, and for investments between a date range entered. You have the option of choosing how you want the data to be ordered by and also if you wish to print the report in Summary or Detail formats.
Irish Cement Salary Data Load Due to the introduction of unlimited contributions, this program has been altered. Once contributions are now loaded onto the Contribution History screen, it is left down to the user to allocate them. They can do this via the "Allocate" option in Members, or use the Contribution Allocation program.
Lookup Lists All Lookup lists throughout Pension Master have been altered to allow the user to order the data, displayed in the lists, by any column. The user can do this by simply clicking on the heading of one of the columns.
Member Labels Member labels have been changed to use the mail merge functionality in Microsoft Word. This allows a greater range of label sizes to be used. A screen has been added to Schemes called Template Path which holds the path of a Microsoft Word template which would be used automatically to generate labels. If no template has been specified then you are guided through the steps required using the Mail Merge Helper in Microsoft Word.
Members Member Address Details Screen. This screen has been adjusted to allow an unlimited number of addresses to be stored for the member. You can search through the addresses by using the new navigation buttons, displayed next to the Salutation field.
Members Divorce Details Screen. This new screen has been added to store the details of any divorces and ex - spouses relating to a member.
Members A new screen has been added to Members called Correspondence History. This records any Mail Merge documents that have been generated for the member.
Members Contribution History. A new menu option called 'Allocate' has been added to this screen. Any unallocated Contributions can be allocated by clicking 'Allocate'. The Contribution Allocation program will then be displayed to the screen.
Members Investment History. A new menu option called 'Post Units' has been added to this screen. Any unposted units can be posted by clicking 'Post Units'. The Units Posting program will then be displayed to the screen.
Members Quotation History. A new menu option called 'Writeback' has been added to this screen. An un-finalised Quotation type can writeback the results by clicking 'Writeback'. The Writeback Quote Results program will then be displayed to the screen.
Members Member Address Details Screen. An extra address line has been added to this screen to allow for easier entry of large addresses.
Member Report/Mail Merge If a field has been selected from the Addresses table, then the argument must be a valid address code. The code must be up to 8 characters long and in uppercase. The second Member Report screen has been split to allow easy entry of report fields which have 2 or more field arguments. The third screen can then be used to amend the field headings if required.

The Pension Increase History/Deferred Pension Increase History fields now have an extra date arument to allow historic data to be extracted.

Member Report Units Two new fields have been added to the screen, 'Not Member Report' and 'Mandatory'. The 'Not Member Report' field should be ticked if the database tables/fields aren't to be accessed through the Member Report and Mail Merge programs.

The 'Mandatory' field should be ticked if the database table (where field contains 'table') is to be used in the General Data Load program. If it is ticked against a database field, then it indicates that data must be loaded into that field during the General Data Load process.

Member Report Units A new field has been added to the screen called 'Writeback Order'.
This field is used in the 'Quotation Type Setup - Writebacks' screen in Categories. The numeric value held in this field is used to order the list of fields displayed in the writeback screen. The writebacks concerned with this new field are Refund, Death, Transfer In and Transfer Out. The order of these fields are defaulted to the order they appear in the corresponding screens in Members
Miscellaneous 'User Defined Arrays' have been renamed to 'User Defined History'.
Schemes Address Details Screen. This screen has been adjusted to allow for an unlimited number of addresses to be stored for a scheme. When a new scheme is added, then the standard 'SCHEME' address code is always set up. The 'SCHEME' address code cannot be deleted, as there should always be a Scheme Address set up.
Schemes Salary Descriptions Screen. This screen has been adjusted to allow for more types of deductible. The deductible can now take the value of:
BM - BSP Married
BS - BSP Single
BW - BSP Widowed
BD - BSP Disabled
L - Lower Earnings Limit
U - User Defined
FO - Formula
N - None

If the deductible takes the value of FO, then a formula code can be selected whose result will then take the value of the deductible.

Schemes User Defined Screen. A new field format of 'FO' has been added, which when used with the entered formula code, will dynamically recalculate the formula when the screen is displayed in Members.
Schemes A Test Scheme field has been added to the Basic Scheme Details screen. This field denotes that the scheme is a test scheme. Test schemes can only contain up to a maximum of 10 members. These members are not included in the total of chargeable members calculated on the Quarterly Fees Print.
Special Codes A group field has now been added to the Special Code screen in members. This allows users to group similar codes together. Members can then be selected using specific codes within a group, or the whole group.
System Parameters Two new fields have been added to the first screen of System Parameters - "Delete Quotation History" and "After n month(s) old". If the first field is ticked then it indicates that member's quotation history details can be deleted. The second field works in conjunction with the first field, and informs the program of how many months old the quotation details must be in order for them to be deleted.
Tax Simplification Amounts NEW. A second screen has been added to the Annual Statutory Amounts program, which allows the user to store Tax Simplification Amounts at system level.

NEW. A screen has been added to Members, which allows the user to store Tax SImplification Amounts at Member level.

Transaction Log Listing NEW program. This program has been developed to allow the user to keep track of any transactions made by members of Pension Master via Pension Master Online.
Defined Contribution System The defined contribution specific parts of Pension Master have been completely revised in this release. There are too numerous changes to mention here but it is all covered in the User Manual and we intend making ourselves available for a day at the time version 7 is installed to go through the main differences and answer any questions you may have.
