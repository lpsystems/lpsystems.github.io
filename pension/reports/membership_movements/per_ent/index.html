Introduction

This program produces a report of members who have joined between two dates entered by the user.

This is especially useful when producing membership movements summaries for a scheme and will show full details of all the individuals who were new entrants during the specified period.

The following options and screen appear when this program is selected:

  • Proceed
  • Exit

Member Selection Screen

Run A Report

Click ‘Proceed’.

Enter the selection criteria required to select the members you wish to output to the report (see Selection Criteria). When the selection criteria has been entered click on ‘OK’ to proceed to the field Entered Date.

You can click on ‘Cancel’ to return to the proceed menu and abort report selection.

ENTERED FIELD -There are three valid entries for this field:

  • Date Entered Company (pm_dec)
  • Date Joined For Life Assurance (pm_djla)
  • Date Joined For Pension (pm_djp)

When this field has been entered a second screen is displayed as follows:

Date Selection Screen

REPORT DATE - The report selection dates default to producing the report for the year immediately prior to today’s date, but can be overwritten by any dates in the format dd/mm/yyyy or ddmmyyyy.

You can click on ‘Cancel’ to exit input of report dates and return to the report menu.

When the value entered in Entered Field is 1 the date field checked is pm_dec - Date Entered Company.

i.e. If pm_dec is >= Report From date and pm_dec <= Report To date the member will be included in the report.

e.g. If a listing of members who entered the company on or between 01/01/92 and 02/02/92 is required then enter 010192 in report from date and 020292 in report to date.

When the value entered in Entered Field is 2 the date field checked is pm_djla - Date Joined For Life Assurance.

i.e. If pm_djla is >= Report From date and pm_djla <= Report To date the member will be included in the report.

e.g. If a listing of members who joined for life assurance on or between 01/01/92 and 02/02/92 is required then enter 010192 in report from date and 020292 in report to date.

When the value entered in Entered Field is 3 the date field checked is pm_djp - Date Joined For Pension.

i.e. If pm_djp is >= Report From date and pm_djp <= Report To date the member will be included in the report.

e.g. If a listing of members who joined for pension on or between 01/01/92 and 02/02/92 is required then enter 010192 in report from date and 020292 in report to date.

When two dates have been entered a message is displayed at the bottom of the screen:

Processing report … Please Wait

When the report has been processed and no records are found to match the selection criteria an error message is displayed:

There are no records to match the selection criteria

When the report has been processed and records are found to match the selection criteria the report continues.

The options for printing are described in the document Printing From Pension Master.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
