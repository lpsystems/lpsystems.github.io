User Account and Valid Password

The first line of security is that someone wanting to use Pension Master must have a valid account set up on the system. This consists of a username or ID together with a password and various other items of user details. These details are then used to determine which options within Pension Master are made available to this user. This enables the System Administrator to customise the look and feel of the software according to the seniority or ability of each user of the system.

Security Level

Each user is assigned a security level from 1 to 99. Security levels must be pre-set using Security Levels. This defines both a security level number and a description pertaining to this security level e.g. can access scheme 0000 but not scheme 0009.

Each member and scheme record on Pension Master is then assigned a corresponding security level from this list of valid security levels and this is matched up against the security level on the current user’s record to determine whether that user has access to any records in that scheme or access to that individual member record. In this way, the System Administrator can easily control access to sensitive records e.g. member records for Director’s of an in-house scheme.

As an extension to this principle, a valid security level can be assigned to each screen in the Member Record on a scheme by scheme basis so that you can limit access to certain screens within member records e.g. Director’s salary records. This is done by using the Member Screen Selection screen in Schemes.

System Administrator Status

A user needs to have a ‘System Administrator’ set to ‘Yes’ in Users in order to access the following Pension Master options:

  • System Parameters
  • Users
  • Menu Options
  • Menus

Only a System Administrator can update/view System factor tables in Factors.

Only a System Administrator can update ‘Allow Manual Update of Units?’ Schemes.

Supervisor Status

If a user has a ‘No’ against ‘Supervisor’ in Users they will be denied access to the following Pension Master options.

  • Schemes
  • Companies
  • Categories
  • Status Codes
  • Brokers
  • Factors
  • Formulae
  • Action Codes
  • Workflow Management
  • Workflow Management Report
  • Codes
  • User Defined Messages
  • Security Levels
  • Annual Statutory Amounts
  • Section 148 Orders
  • CEP Factors
  • MLI/RPI Factors
  • Locks
  • Audit Trail Listing

If the user has a ‘V’ against ‘Supervisor’ in Users they will be allowed access to the above options, but will be limited to view only and not view and update. This disables any menu items in the above options which would allow the user to add/update/delete any item of data. The exceptions to this are Workflow Management Report and Audit Trail Listing which are reports only and do not allow any user input other than running the report.

If the user has a ‘Yes’ against ‘Supervisor’ in Users they will be allowed to:

  • Fully access the above options.
  • Delete/update any other user’s diary using Diary File Maintenance.
  • Undo posted units in Post Units.
  • Delete a member record.
  • Un-finalise quotes in Members.
  • Override quotation results in Members.

Update Member Records Status

If a user has a ‘No’ against ‘Update Member Records’ in Users they will not be allowed to add/update/delete member records in Members.

Otherwise the user will be granted full access to the member records, subject to any other security procedures described herein.

Restrict Access to Individual Schemes

It is possible to restrict access to particular schemes via use of the User Access Details screen in Schemes. There are three types of access level:

  • All users have access to the scheme.
  • All members of a particular workgroup have access to the scheme. See Workgroups.
  • User level access. Here you define particular users via their user ID (as set up in Users) so as to restrict access to a scheme on an individual user level.

If any user who does not have access to a scheme tried to select a record from that scheme, the only record header will be displayed e.g. in the member file, only the members name, scheme, company, category and status details will be displayed to the screen. Only schemes to which the user has access will be displayed in all lookup Scheme lists, available by clicking on the lookup icon.
